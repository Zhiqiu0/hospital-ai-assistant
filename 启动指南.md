# MedAssist 本地启动指南

## 前提：确认 PostgreSQL 已启动

说明：仓库当前没有内置 `docker-compose.yml`。如果你已经用 Docker 单独启动了 PostgreSQL / Redis，保持容器运行即可；本项目本地启动只需要手动启动前后端。

数据库连接信息（已配置在 `.env`）：
- 地址：`localhost:5432`
- 数据库名：`hospital_ai`
- 用户名：`hospital_user`
- 密码：`hospital_pass_2024`

如果数据库还没创建，先执行：
```sql
CREATE USER hospital_user WITH PASSWORD 'hospital_pass_2024';
CREATE DATABASE hospital_ai OWNER hospital_user;
```

## 第一次启动（只需做一次）

在 `medassist/backend` 目录下，激活虚拟环境后初始化数据库：

```bash
cd d:/Projects/VsCodeProjects/cc_test/medassist/backend
venv/Scripts/activate
PYTHONUTF8=1 python init_db.py
```

成功后会输出：
```
[OK] 数据库表创建完成
[OK] 默认数据插入完成

===================================
  默认账号：
  管理员  - admin / admin123456
  测试医生 - doctor01 / doctor123
===================================
```

## 每次启动（开两个终端）

### 终端 1 — 启动后端

```bash
cd d:/Projects/VsCodeProjects/cc_test/medassist/backend
venv/Scripts/activate
PYTHONUTF8=1 uvicorn app.main:app --reload --port 8000
```

看到以下输出表示启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
```

> API 文档地址：http://localhost:8000/docs

### 终端 2 — 启动前端

```bash
cd d:/Projects/VsCodeProjects/cc_test/medassist/frontend
npm run dev
```

看到以下输出表示启动成功：
```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5174/
```

## 常见问题

**后端启动报编码错误 / 中文乱码**
→ 确保命令前加了 `PYTHONUTF8=1`

**提示 `ModuleNotFoundError`**
→ 确认已执行 `venv/Scripts/activate` 激活虚拟环境

**前端提示接口 404 / 502**
→ 检查后端是否已启动，端口是否是 8000

**数据库连接失败**
→ 检查 PostgreSQL 是否在运行，`hospital_ai` 数据库是否已创建

**Docker 已启动但项目仍连不上数据库**
→ 检查容器端口是否映射到 `localhost:5432`，并确认 `.env` 中 `DATABASE_URL` 与实际容器配置一致
