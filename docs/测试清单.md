## 🔧 常见故障排查

| 现象 | 原因 | 处置 |
|---|---|---|
| 患者档案卡片里字段全空（明明李四之前有值） | 后端没 reload，profile JSONB 迁移没跑 | 重启后端，看启动日志确认迁移 SQL 执行 |
| 患者档案卡里月经史还在 | 前端老 chunk 缓存 | F12 → Application → Storage → Clear site data |
| 「✓ 仍准确」按钮看不到 | 字段为空 / 距今 ≤ 7 天属于"新鲜"不显示按钮 | 设计如此，新鲜数据不打扰医生 |
| 「办理出院」按钮没出现 | 没选中患者 | 先从左侧病区点一个患者 |
| 出院按钮报 403"只有主治医生可办理" | 用了别人的账号 | 用接诊创建者的账号（李四是 zz） |
| 续接诊点已签发还是弹"请去历史病历"老提示 | 前端老 chunk | Clear site data 刷新 |
| 出院后病区列表没刷新 | Redis my_encounters 缓存没失效（理论上自动会清） | 手动刷新一下页面，或清 Redis：`redis-cli DEL "encounter:my:<doctor_id>"` |
| 字段时间戳显示"X 年前确认"颜色全是灰色 | 距今 ≤ 7 天才是灰；本轮迁移把所有老字段时间戳设为迁移那天，所以前几天看都是灰 | 等 8 天就会变褐色 |
