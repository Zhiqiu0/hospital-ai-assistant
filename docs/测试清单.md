## 🔴 S1：PACS 鉴权漏洞修复 — **必须完整测**

> 后端：thumbnail / dicom / frames 三端点接入 `assert_patient_access`
> 前端：缩略图改走 `fetch + blob URL`（携带 Authorization 头）

### A. 影像科医生正常路径
- [ ] 用**影像科医生**账号登录 → 进 PACS 工作台
- [ ] 上传一个 DICOM 包（ZIP/RAR）
- [ ] 缩略图列表能正常显示**每一张图**（不应该出现"加载失败"占位符）
- [ ] 点缩略图打开 `DicomViewer`，画面正常渲染
- [ ] 拖**窗位（WC）/ 窗宽（WW）** 滑块，画面应实时更新
- [ ] DevTools Network 面板：thumbnail 请求 Header 应有 `Authorization: Bearer ...`
- [ ] 控制台**不应有 401/403** 报错

### B. 跨患者权限拒绝（核心安全验证）
- [ ] 用**普通医生**账号登录（这个医生没有接诊过 PACS 上那个患者）
- [ ] 试图访问刚才那个 study 的缩略图：手动构造 URL 或访问对应患者的影像页
- [ ] 应该返回 **403 / "无权访问该患者的病历档案"**，缩略图位置显示"加载失败"

### C. 普通医生有权限路径
- [ ] 用**普通医生**账号给某患者建一次接诊（让自己成为该患者的接诊医生）
- [ ] 用影像科账号给该患者上传影像
- [ ] 切回普通医生账号，去患者档案/历史影像页面，应该能正常看到缩略图
