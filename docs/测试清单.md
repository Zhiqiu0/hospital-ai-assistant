## 🔴 B：auto-save 编辑器（输入到一半也不丢）

**为什么有这个**：医生在病历编辑器里手动改文字时，5 秒防抖自动保存到后端。浏览器崩溃 / 不小心关标签 / 网络断都最多丢最后 5 秒的输入。

1. [ ] 接诊后 AI 生成病历 → 在中间病历编辑器里**手动加几个字**（比如末尾敲"测试 auto save"）
2. [ ] 保持那个状态 5-6 秒不动 → 打开浏览器 F12 Network 面板 → 应该看到一个 `POST /medical-records/auto-save-draft` 请求 status 200
3. [ ] **强制刷新页面**（Ctrl+F5）→ 编辑器内容应该还在（含你刚才加的字）
4. [ ] 再加几个字 → 等 5 秒 → 第二次 auto-save 触发 → Network 看到第二个 `auto-save-draft` 请求
5. [ ] **logout 再登回来** → 进同一接诊 → 编辑器内容还在（含两次输入）

**多设备冲突测试**（可选，难手动触发）：
- [ ] 在 Chrome 和 Edge 同时登录同一账号 → 两个浏览器都打开同一接诊
- [ ] Chrome 编辑 → 等保存
- [ ] Edge 改一个不同位置 → 等 5 秒 → 应该弹 toast "病历已被其他设备修改"

**网络断测试**（可选）：
- [ ] F12 Network 面板 → 改成 "Offline"
- [ ] 编辑器输入 → 等 5 秒 → 看到 `auto-save-draft` 请求失败（这正常）
- [ ] Network 改回 "Online" → 切换接诊或刷新 → 之前的输入应被自动补传到后端

## 🟢 D：AI 病历生成（L3 治本核心）

**为什么重点测**：之前 LLM 偶尔输出"切诊：脉弦"、"望诊：神清，舌淡红"等格式偏差导致 QC 误报。本轮把 LLM 输出从"自由文本"改成"JSON + 后端模板拼装"，根除偏差。

**门诊（中医病历）**——`doctor01` 登录，初诊登记 → 填问诊（含舌象/脉象/中医诊断）→ 点"AI 一键生成"：

- [ ] 章节齐全：【主诉】【现病史】【既往史】【过敏史】【个人史】【体格检查】【辅助检查】【诊断】【治疗意见及措施】
- [ ] 【体格检查】**第一行就是生命体征**：`T:36.5℃ P:78次/分 R:18次/分 BP:120/80mmHg` 这样的格式
- [ ] 【体格检查】下应有 **5 个明确子行**：望诊：xxx / 闻诊：xxx / 切诊·舌象：xxx / 切诊·脉象：xxx / 其余阳性体征：xxx
- [ ] **绝对不应该出现**：`切诊：xxx`（合并写法）；独立的【望诊】/【舌象】/【脉象】/【中医诊断】章节
- [ ] 【诊断】里应有 `中医诊断：感冒 — 风寒束表证` 合并行 + `西医诊断：xxx` 子行
- [ ] 【治疗意见及措施】下 4 个子行：治则治法 / 处理意见 / 复诊建议 / 注意事项
- [ ] **空字段处理**：医生没填的字段应原样写 `[未填写，需补充]`，绝对不能 LLM 自己编造（合规底线）

**急诊**——切到急诊接诊 → 填问诊 → AI 生成：
- [ ] 章节：【主诉】【现病史】【既往史】【过敏史】【体格检查】【辅助检查】【诊断】【急诊处置】【患者去向】
- [ ] **不应有**【个人史】章节、不应有中医四诊子行
- [ ] 【体格检查】首行也是生命体征 `T:... P:... R:... BP:...`

**住院记录**——`zz` 住院端 → 新建住院接诊 → 填问诊（含 7 项专项评估）→ AI 生成入院记录：
- [ ] 章节齐全：主诉/现病史/既往史/个人史/婚育史/（女性才有月经史）/家族史/病史陈述者/专项评估/体格检查/辅助检查（入院前）/入院诊断
- [ ] 【专项评估】下应有 **7 个子行**：`· 当前用药：xxx` / `· 疼痛评估（NRS评分）：xxx` / `· 康复需求：xxx` / `· 心理状态：xxx` / `· 营养风险：xxx` / `· VTE风险：xxx` / `· 宗教信仰/饮食禁忌：xxx`
- [ ] 男性患者**不应**出现【月经史】章节
- [ ] 住院【体格检查】首行也是生命体征

## 🔴 E：QC 修复操作（取消快照 bug + 行级写入）

**前置**：让医生录入故意留几项空（如不填舌象/脉象/生命体征），AI 生成后点 "AI 质控"，等 QC 卡片出现再开始测。

**行级写入互不冲突**：
- [ ] 找到"舌象未填"卡片 → 点 AI 修复 → 编辑器自动填好建议 → 点「写入病历」→ 病历里"切诊·舌象：xxx"行被填上
- [ ] 接着点"脉象未填" → AI 修复 → 写入病历 → "切诊·脉象：xxx" 也填上，**舌象保留**
- [ ] 接着点"生命体征未填" → AI 修复 → 写入病历 → 体格检查段第一行变成完整 T:/P:/R:/BP:，**舌象 + 脉象都还在**
- [ ] 反过来：先点生命体征写入，再点舌象写入，最后点脉象写入 → 三者都在，互相不覆盖

**取消快照 bug 修复（关键！）**：
- [ ] 接着上面的状态 → 点已写入的"舌象"再点一次（取消） → 舌象一行回到 `[未填写，需补充]`，**脉象 + 生命体征都还在**（之前的 bug：会把脉象 + 生命体征都冲掉）
- [ ] 同样：先写望诊 → 写闻诊 → 取消望诊 → 望诊回到占位符，**闻诊保留**

**QC 不再误报**（L3 治本副产品）：
- [ ] 把所有字段都正确填写后再做一次 AI 质控 → **不应再看到** "舌象未填" / "脉象未填" / "望诊未填" / "闻诊未填" / "生命体征未填" 这些误报

## 🔧 常见故障排查

| 现象 | 处置 |
|---|---|
| AI 生成转圈很久才出文本 | 现在是"等几秒整段出现"而不是逐字打字机，**这是预期** |
| 拼音搜索搜不到老患者 | 部署完后跑过 `alembic upgrade head` 没？migration 会回填存量患者拼音 |
| QC 报"舌象未填"但医生明明填了 | 看病历是否含"切诊：xxx"（合并）行，应是"切诊·舌象：xxx"。新版 prompt 已修，老存量病历可能还有 |
| 取消 QC 修复把别的字段也冲掉了 | 旧版 bug，部署最新代码（本轮已修复） |
| logout 重登病历/质控/追问空白 | 是 bug。检查 Network 看 `/encounters/:id/workspace` 接口返回是否含 `latest_qc_issues` / `latest_ai_suggestions` / `active_record.content` 字段 |
| auto-save 没触发 | F12 Network 看是否有 `auto-save-draft` 请求；输入后等 5 秒触发 |
| auto-save 弹"病历已被其他设备修改" | 多设备同时编辑了同一接诊，按提示刷新 |
| Orthanc 影像看不了 | `docker logs orthanc` 看是否启动报错；多半是 .env 缺 ORTHANC_PASSWORD |
| 部署 docker compose up 报 "required variable missing" | .env 缺必填变量。不是 bug，是设计 — 见 F 章节 |
