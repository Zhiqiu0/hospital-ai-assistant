# MediScribe 项目开发总结

> 文档版本：V1.0 | 更新日期：2026-03-19

---

## 一、项目概述

**MediScribe** 是一个面向医院的临床 AI 辅助系统，旨在帮助医生高效完成问诊记录、病历生成、质量控制和病历签发全流程。系统包含医生工作台和管理后台两大模块。

---

## 二、技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | React 18 + TypeScript + Ant Design 5 + Vite + Zustand |
| 后端 | FastAPI + SQLAlchemy 2.0 async + asyncpg |
| 数据库 | PostgreSQL |
| AI 服务 | DeepSeek API（via OpenAI SDK，支持 SSE 流式输出） |
| 认证 | JWT（python-jose） |
| 运行环境 | Python venv + Node.js |

## 五、已实现功能

### 5.1 认证系统
- JWT 登录 / 登出
- 角色区分：医生（doctor）、管理员（admin）
- 路由守卫：未登录自动跳转登录页
- 分裂屏登录页（左侧品牌介绍 + 右侧登录表单）

### 5.2 医生工作台

#### 问诊面板
- 结构化填写：主诉、现病史、既往史、过敏史、体格检查、初步印象
- 保存到数据库（与接诊记录关联）
- 切换接诊后自动重置表单

#### 病历编辑器
- 富文本编辑区域，支持手动输入
- **一键生成**：基于问诊内容 SSE 流式生成病历
- **AI 质控**：检查病历质量，列出问题清单
- **润色**：对病历内容进行 SSE 流式润色优化
- **出具最终病历**：
  - 展示质控通过状态
  - 无接诊记录时弹出患者信息表单（姓名 + 性别 + 年龄），自动创建患者档案和接诊记录
  - 医生阅读确认并勾选后签发
  - 签发后病历变为只读，按钮全部禁用

#### AI 建议面板（右侧，360px）
共三个 Tab：

**追问建议**
- AI 根据主诉生成追问问题列表
- 每个问题附带 2-4 个具体选项按钮
- 医生点选后自动将答案记录到现病史
- 支持"获取更多追问"按钮继续生成
- 诊断引导区：AI 给出初步诊断建议，一键写入初步印象字段

**检查建议**
- 根据病历内容实时推荐检查项目（1.2s 防抖）
- 分优先级展示

**质控提示**
- 展示每条质控问题
- 每条问题支持：AI 生成修复建议 / 手动编辑 / 写入病历

#### 历史病历
- 顶部"历史病历"按钮打开抽屉
- 展示自己所有已签发病历
- 点击可查看完整内容

#### 新建接诊
- 顶部"新建接诊"按钮，填写患者信息后创建接诊记录
- 创建后工作台状态自动重置

### 5.3 管理后台

| 模块 | 功能 |
|------|------|
| 系统概览 | 彩色统计卡片（用户数、接诊量、病历数、AI 调用数）+ 快捷导航 |
| 用户管理 | 新建、编辑、启用/停用用户，角色分配 |
| 科室管理 | 科室增删改查 |
| 质控规则 | 规则配置管理 |
| Prompt 管理 | AI Prompt 的维护与版本管理 |
| 数据统计 | 图表展示系统使用情况 |
| 病历管理 | 查看所有已签发病历，支持按患者/医生搜索 |

## 六、后端接口清单

| 分类 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 认证 | POST | `/auth/login` | 登录获取 Token |
| 问诊 | POST | `/inquiries/save` | 保存问诊信息 |
| 接诊 | POST | `/encounters/quick-start` | 快速创建患者+接诊 |
| 病历 | POST | `/medical-records/quick-save` | 签发保存病历 |
| 病历 | GET | `/medical-records/my` | 获取当前医生的病历 |
| AI | POST | `/ai/quick-generate` | 流式生成病历 |
| AI | POST | `/ai/polish` | 流式润色病历 |
| AI | POST | `/ai/inquiry-suggestions` | 获取追问建议 |
| AI | POST | `/ai/exam-suggestions` | 获取检查建议 |
| AI | POST | `/ai/quick-qc` | 快速质控 |
| AI | POST | `/ai/qc-fix` | AI 生成质控修复建议 |
| AI | POST | `/ai/diagnosis-suggestion` | 诊断引导建议 |
| 管理 | GET/POST | `/admin/users` | 用户管理 |
| 管理 | GET/POST | `/admin/departments` | 科室管理 |
| 管理 | GET | `/admin/records` | 所有病历（管理员） |

## 七、本地开发启动方式

### 启动后端

```powershell
cd backend
.\venv\Scripts\activate
$env:PYTHONUTF8=1; uvicorn app.main:app --reload --port 8000
```

### 启动前端

```powershell
cd frontend
npm run dev
```

### 访问地址

| 地址 | 说明 |
|------|------|
| http://localhost:5174 | 前端页面 |
| http://localhost:8000/docs | 后端 API 文档（Swagger） |

### 停止服务

在对应终端按 `Ctrl+C`。

### 查看端口占用

```powershell
netstat -ano | findstr ":8000"
netstat -ano | findstr ":5174"
```
