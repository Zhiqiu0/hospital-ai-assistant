# 医院临床接诊智能助手系统 PRD（一期优化版）

> 文档版本：V1.1 | 产品阶段：一期 MVP | 更新日期：2026-03-18

---

## 一、项目概述

### 1.1 背景与痛点

| 痛点 | 影响 |
|------|------|
| 问诊不完整，关键病史遗漏 | 误诊风险，患者满意度低 |
| 病历书写耗时，重复劳动多 | 医生效率低，接诊量受限 |
| 病历缺项、逻辑不一致 | 终末质控返工成本高 |
| 医保审核风险高 | 医院经济损失 |

### 1.2 解决方案

建设一套面向**医生端 Web**的智能辅助系统，核心能力三项：

```
问诊辅助  →  病历生成  →  合规质控
```

### 1.3 一期明确不做的事

- 自动下达检查申请单、处方
- 自动给出正式诊断结论
- 实时语音转写
- 患者端功能
- 多医院 SaaS 平台

## 二、用户与场景

### 2.1 用户角色

| 角色 | 核心诉求 | 使用频率 |
|------|---------|---------|
| **医生**（核心） | 快速生成病历、减少遗漏、降低文书负担 | 每日高频 |
| 科主任/质控人员 | 查看质控问题分布、跟踪改善趋势 | 每周 |
| 管理层/信息科 | 系统配置、权限管理、效果评估 | 按需 |

### 2.2 核心使用场景

**场景一：门诊接诊**
```
医生输入主诉 → AI提示追问项 → 补充信息 → 一键生成门诊病历 → 审核保存
```

**场景二：住院病历书写**
```
录入患者信息 → AI生成入院记录草稿 → 质控扫描 → 医生修订提交
```

**场景三：病历提交前质控**
```
病历完稿 → 点击AI质控 → 查看风险问题列表 → 逐项修改/忽略 → 提交
```

## 三、功能模块详情

### 3.1 模块总览

```
医生工作台（主界面）
├── 接诊问诊助手（AI追问）
├── 病历生成助手（一键生成/续写/润色）
├── 检查建议助手
└── 合规质控助手（完整性/规范性/医保风险）

后台管理系统
└── 日志与统计系统
```

---

### 3.2 模块一：医生工作台

**页面布局（三栏）**

```
┌──────────────┬──────────────────┬──────────────┐
│   左侧输入区  │    中间编辑区     │  右侧建议区   │
│              │                  │              │
│ • 患者基本信息│ • AI生成病历草稿  │ • 追问建议    │
│ • 主诉       │ • 可直接编辑      │ • 检查建议    │
│ • 现病史     │ • 支持局部重写    │ • 质控风险    │
│ • 既往史     │ • 支持润色/续写   │              │
│ • 过敏史     │                  │              │
│ • 个人史     │                  │              │
│ • 体格检查   │                  │              │
│ • 初步印象   │                  │              │
└──────────────┴──────────────────┴──────────────┘
```

**顶部固定栏**：患者信息 + 就诊类型（门诊/住院）+ 保存/生成/质控按钮

**交互要求**
- 重点按钮固定可见
- AI生成中显示加载状态
- 调用失败有明确提示，不丢失已输入数据
- AI生成内容标注"仅供医生参考，须经医生审核确认"

---

### 3.3 模块二：接诊问诊助手

**功能逻辑**

输入：主诉 + 现病史 + 科室信息
输出：追问建议列表（高优先级 / 一般优先级）+ 危险信号提醒

**具体能力**

| 能力 | 说明 |
|------|------|
| 主诉识别 | 提取关键症状，判断症状类别 |
| 智能追问 | 推荐追问项（病程、诱因、伴随症状、缓解/加重因素等） |
| 危险信号提示 | 胸痛、呼吸困难、黑便、昏厥、高热等高危症状优先提醒 |
| 特殊人群提醒 | 老年/儿童/孕妇额外追加提示 |
| 问诊完整度 | 提示当前仍缺少的关键问诊维度 |

**交互**
- 非阻断展示，不影响医生主流程
- 医生可操作：已问 / 忽略 / 插入记录
- 支持刷新建议

**验收标准**
- 主诉输入后 **3秒内** 返回建议
- 建议按优先级排序，区分高优/一般

---

### 3.4 模块三：病历生成助手

**支持病历类型（一期）**

| 类型 | 说明 |
|------|------|
| 门诊病历 | 完整版 |
| 入院记录 | 简版 |
| 首次病程记录 | 简版 |

**生成内容**：主诉、现病史、既往史、过敏史、个人史、体格检查模板、初步诊断描述

**四种操作模式**

| 模式 | 触发时机 | 说明 |
|------|---------|------|
| 一键生成 | 输入基本信息后 | 生成完整病历草稿 |
| 续写 | 医生已写部分内容 | 补全未完成段落 |
| 润色 | 草稿已完成 | 口语→书面、规范化、消除重复、优化时序 |
| 补全 | 质控后 | 识别缺失要素并填充 |

**验收标准**
- 完整输入基本问诊信息后，**5-10秒内** 输出结构清晰草稿
- 支持至少 80% 常规门诊场景

---

### 3.5 模块四：检查建议助手

**输出内容**

| 类别 | 说明 |
|------|------|
| 基础建议 | 常规必查项 |
| 排查性建议 | 鉴别诊断相关 |
| 高风险补充建议 | 重要但易遗漏的检查 |

**边界**：仅做建议提示，不自动下单，明确声明仅供参考

---

### 3.6 模块五：合规质控助手

**四类质控维度**

#### 完整性质控（规则引擎）
检查项：主诉缺失或过长 / 现病史要素不完整 / 既往史缺失 / 过敏史未填 / 体格检查缺项 / 诊断依据不足

#### 规范性质控（LLM）
检查项：表述口语化 / 病程描述不清 / 时间线混乱 / 内容重复堆叠 / 诊断表述不规范

#### 逻辑一致性质控（LLM）
检查项：主诉与现病史不一致 / 现病史与诊断方向不一致 / 检查诊断治疗衔接不足

#### 医保风险提示（规则 + LLM）
检查项：记录不支持检查合理性 / 记录不支持治疗必要性 / 用药依据不充分 / 诊断与治疗强度不匹配

**风险分级展示**

| 级别 | 颜色 | 处理建议 |
|------|------|---------|
| 🔴 高风险 | 红色 | 提交前必须处理 |
| 🟡 中风险 | 黄色 | 建议完善 |
| 🟢 低风险 | 绿色 | 可酌情处理 |

**每条问题包含**：问题类型 + 风险等级 + 问题说明 + 修改建议 + 跳转到病历对应位置

## 四、技术方案

### 4.1 架构分层

```
前端层        React + Ant Design
  ↓
服务层        Python FastAPI
  ├── 用户权限服务
  ├── 病历生成服务
  ├── 追问建议服务
  ├── 检查建议服务
  ├── 质控服务
  └── 日志统计服务
  ↓
AI中台层
  ├── 模型网关（对接大模型API）
  ├── Prompt 编排与版本管理
  ├── 输出结构化解析
  └── 规则引擎（完整性/医保规则）
  ↓
数据层
  ├── PostgreSQL（业务数据）
  ├── Redis（缓存/会话）
  └── 文件存储（日志/模板）
  ↓
接口层
  └── HIS读取接口（一期仅读：医生信息/患者基础信息/就诊编号）
```

### 4.2 AI策略：规则 + 模型分工

| 任务 | 用规则 | 用模型 |
|------|--------|--------|
| 必填项检查 | ✅ | |
| 字数/格式校验 | ✅ | |
| 医保关键词识别 | ✅ | |
| 病历文本生成 | | ✅ |
| 追问建议生成 | | ✅ |
| 逻辑一致性判断 | | ✅ |
| 规范性润色 | | ✅ |

> 原则：高风险内容规则兜底，模型负责理解与生成

### 4.3 一期HIS对接范围（保持最小）

**对接**：医生身份信息 + 患者基础信息 + 就诊编号
**不对接**：病历自动回写、检查单下单、处方联动

## 八、风险与应对

| 风险 | 应对方案 |
|------|---------|
| 医生接受度低 | 简化交互、先选配合度高的科室试点、辅助而非强制 |
| 模型输出不稳定 | 规则兜底、提供重试与局部重写、持续调优 |
| HIS对接延误 | 一期先独立Web运行，仅做必要数据读取 |
| 医保规则维护难 | 规则配置化，管理后台可视化维护 |
